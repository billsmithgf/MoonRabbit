About solving the problem of margin-top overlap

Author：Bill Lin

If the parent element contains a floating element, and the parent element does not set the height and width, then its height will collapse to zero, which is the so-called "height collapse".

If the parent element contains a background or border, the overflowing element is not considered part of the parent element.

Solution 1: Give the parent element a fixed height,But the disadvantage of this method is: not flexible enough to adapt.

Solution 2: Add attribute overflow: hidden; to the parent element The advantage is that it is simple. Open the BFC directly. The disadvantage is that if a position sub-element exceeds the container, it will be cropped.

Solution 3: Add an empty div at the end of the child element and set the style below. Advantages: all browsers support, and container overflow will not be clipped; the disadvantage is that it does not look good.

Option 4: Perfect pseudo-element clearing floating method.Add before and after pseudo-elements before or after the parent element, content: ‘’, clear: both; display: block.
